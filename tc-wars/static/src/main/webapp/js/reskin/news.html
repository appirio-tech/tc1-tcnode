    General TopCoder News Announcing a New DTN Challenge!

    NASA has just launched a new contest that is part of the Disruption Tolerant Networking (DTN) challenge. DTN is an approach to computer network architecture that seeks to address the technical issues in heterogeneous networks that may lack continuous network connectivity. The contests we have run so far gathered and organized the information on DTNs and reduced the problem to be solved to a special form of a theoretical abstraction used in distributed computing known as the Byzantine Generals’ problem. The new contest asks you to study the output of previous contests and to create a game plan that specifies which contests need to be run in order to successfully solve the given problem. If you love logic puzzles, algorithms and figuring out complex problems, then this is the contest for you! To register, please visit here.

    Algorithm Competition News Algorithm Competitions Update: custom time/memory limits

    Starting from SRM 600 TopCoder will be switching to custom time and memory limit settings per problem. We’ve added a separate section “Limits” to the problem statement where both time and memory limits are specified.

    Despite that we are able to use any limits, we intend to keep having the same settings for almost all problems as it’s done now. The settings will be 2 seconds and 256 megabytes. Other settings will be used only if the problem requires it and can’t be done with the new default settings. The statement of any such problem will start with a non-standard limits warning message.

    All old problems will continue having 2 seconds and 64 megabytes.
